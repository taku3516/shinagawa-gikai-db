# 質問・答弁要約（qaSummaries）の作成ルール

一般質問・代表質問の「質問の要約」「答弁の要約」を作成・生成するときの共通ルール。
新しい年（過去年の遡り実装を含む）を追加するときは必ずこのルールに従い、
最後に「機械チェック」を実行してから公開する。

## スキーマ

質問エントリ（`questions[]`）に `qaSummaries` 配列を持たせる。

```json
{
  "title": "発言項目名（例: 防災について）",
  "question": "質問の要約（1〜2文・完結した文）",
  "answer": "答弁の要約（1〜2文・完結した文）"
}
```

## 文体ルール

1. **必ず完結した文にする**。文末は「。」で終わる。途中で切れた抜粋を置かない
   - 抜粋が長すぎる場合は文の区切り（。）で切る。文の途中でしか切れない場合は
     「…などについて質問し、区の見解を尋ねました。」「…などと答えました。」で文として完結させる
2. **質問は間接話法**で書く。議員の発言をそのまま接ぎ木しない
   - NG: 「議員は、〜についてお聞かせください。」（一人称の命令形が残る）
   - OK: 「議員は、〜についての見解を尋ねました。」「〜の検討を求めました。」
   - 文末の変換表: お聞かせください→説明を求めました ／ ご見解を伺います→見解を尋ねました ／
     検討いただきたい→検討を求めました ／ すべきです→べきだと主張しました
3. **答弁は「区側は、」で始め、です・ます体**で書く
   - 「私からは」「お答えいたします」などの前置きだけの文は採用しない（実質的な内容がある文を選ぶ）
   - 冒頭の接続詞（初めに、まず、次に）は「区側は、」の直後に置かない
4. **長さの上限**: 質問190字・答弁200字（文の区切りで収める）
5. **重複表現の禁止**: 「「〜について」について」のような二重表現を作らない
   - 定型文を使う場合、titleが「〜について」で終わるならその部分を除いてから当てはめる
   - titleに鍵括弧が入れ子になっている場合も同様（貪欲マッチで末尾の「について」を除去）
6. **文頭の断片禁止**: 語の途中や助詞から始まる抜粋は、最初の完全な句から始める

## 内容が取れない場合の標準文言（フォールバック）

- 質問: 「「{title}」について、現状の認識と区の対応を質問しました。」
- 答弁: 「答弁の全文は会議録を参照してください。」

フォールバックは正直に使う。無理に文をでっち上げない。

## 出典と免責（必須）

- 各質問エントリの `links` に、該当日の会議録（`Template=document&Id=N`）または
  会議録検索へのリンクを必ず含める
- 表示側の免責「※質問・答弁の要点を本サイトで要約しています。正確な発言は公式の会議録をご確認ください。」
  が出ることを確認する（qaSummaries があれば自動表示される）

## 生成手順（会議録からの機械抽出の場合）

1. 会議録本文から発言スパンを切り出す（`〔○○君登壇〕`マーカー。役職者
   （区長・教育長・次長・部長・議長・委員長・監査・室長・所長・管理者）は答弁側）
2. 発言者ヘッダ（`N:◯区長（…）`）は必ず除去してから文抽出する
3. 質問がある日だけを対象にする（議事日程に「質問」の記載がある日。
   最終日の委員長報告・討論の登壇者を質問者に混ぜない）
4. テーマは質問文の「初めに／次に／第一に／まず初めは」等の列挙から抽出
5. 答弁の照合はテーマの先頭8字→6字→4字→末尾5字の順で試す。
   前置き文（60字未満で「お答え」を含む文）はスキップして次の実質文を採る

## 機械チェック（公開前に必ず実行）

```python
# 全データファイルに対して以下がすべて0件であること
PATTERNS = {
    "文末未完結":       r'"(?:question|answer)": "[^"]*[^。"]"',
    "二重について":     r'について」について',
    "一人称の接ぎ木":   r'(?:お聞かせ|お示し|お答え)ください。"',
    "例えば始まり":     r'"question": "議員は、例えば、',
    "接続詞始まり答弁": r'"answer": "区側は、(?:初めに|まず|次に)、',
    "長すぎる要約":     None,  # question>190字 / answer>200字 をコードで判定
}
```
